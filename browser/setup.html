Media Processor Setup

Complete these steps to enable browser integration

Step 1: Install Media Processor Desktop Application

Make sure you have the Media Processor desktop application installed and running on your computer.

✓ Extension Installed: You already have the browser extension installed!

Step 2: Setup Protocol Handler

The protocol handler allows the browser to communicate with your desktop application.

Windows
macOS
Linux

Windows Setup:

  1. Run the Media Processor application as Administrator (right-click → "Run as administrator")
  2. The application will automatically register the protocol handler
  3. If prompted by Windows, click "Yes" to allow the registration
  4. Restart your browser after first setup
Note: You may need to run the application as Administrator the first time to register the protocol handler.

macOS Setup:

  1. Run the Media Processor application
  2. The application will create the necessary protocol handler files
  3. You may need to approve the application in System Preferences → Security & Privacy
  4. Test the protocol by clicking a link below

Linux Setup:

  1. Run the Media Processor application
  2. The application will create a .desktop file for protocol handling
  3. Run: xdg-mime default mediaprocessor.desktop x-scheme-handler/mediaprocessor
  4. Restart your browser

Step 3: Test the Integration

Click the button below to test if the protocol handler is working:

Test Protocol Handler

This will try to send a YouTube URL to your Media Processor application.

Step 4: How to Use

Once setup is complete, you can send URLs to Media Processor in several ways:

  • Extension Button: Click the Media Processor icon in your browser toolbar
  • Context Menu: Right-click on any page and select "Send to Media Processor"
  • Bookmark: Use the provided bookmark on any media page

Troubleshooting

If the protocol handler isn't working:

  • Make sure Media Processor is running
  • Try restarting your browser
  • On Windows, try running Media Processor as Administrator once
  • Check that no firewall is blocking the application

Alternative method - Copy and paste:

If direct protocol handling doesn't work, you can copy protocol links and paste them into your address bar:

mediaprocessor://https://example.com/video
